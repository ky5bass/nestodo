# GitHub 共通運用ルール

## 参照とレビュー投稿

- Issue を参照するときは、仕様変更や追加要件を見落とさないように本文とコメントの両方を取得する
- PR 番号または PR URL を指定したレビュー依頼では、対象 PR へのレビューコメント投稿を既定動作とする

## AI エージェントの表示

Kiro、Codex、Claude Code が Issue、Issue コメント、PR、PR の通常コメント、インラインレビューコメント、レビュー返信など、本文を持つ GitHub リソースを作成するときは、本文末尾に次の署名を付ける。GitHub 上の投稿者アカウントだけでは、同じアカウントを使う AI エージェントを区別できないため、可視署名と機械判定用マーカーを併用する。

```markdown
<sub>Generated by Kiro</sub>
<!-- ai-agent:kiro -->
```

```markdown
<sub>Generated by Codex</sub>
<!-- ai-agent:codex -->
```

```markdown
<sub>Generated by Claude Code</sub>
<!-- ai-agent:claude-code -->
```

- 人間が作成したリソースには署名を追加しない
- 他者の文面を引用する場合も、末尾の署名は実際に投稿操作を行ったエージェントを示す
- 既存の識別子コメントがある場合は、その直前に可視署名と `ai-agent` マーカーを置く
- タイトル、ラベル、close、resolve など本文を持たない操作は、関連する本文付きリソースまたは操作報告コメントで作成・実行主体を示す

## レビュー指摘への対応

指摘へ対応するエージェントは、すべての指摘スレッドへ次のいずれかを返信する。

- 対応した場合: 対応内容、commit SHA、検証結果
- 対応しない場合: 対応しない判断、具体的な理由、根拠

指摘は提案として評価し、妥当性を確認せず機械的に採用しない。ただし、対応しない指摘を無言で放置してはならない。

Claude Code は、修正または見送り理由を妥当と判断したスレッドだけを解決する。未対応、検証不足、理由が不十分なスレッドは解決しない。

## コメント・本文の渡し方

`gh` CLI で本文を渡すときは `--body` ではなく `--body-file` を使用する。zsh ではダブルクォート内の `!` がヒストリ展開として解釈され `event not found` エラーになるため、本文は一時ファイル経由で渡す。
