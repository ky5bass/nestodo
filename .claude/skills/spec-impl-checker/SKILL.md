# Spec と実装の整合性確認

Kiro が spec を生成し Codex が実装した場合の確認観点（ステップ6）:

1. **要件カバレッジ**: `requirements.md` の受け入れ基準が実装・テストで満たされている
2. **設計の遵守**: `design.md` の設計判断・インターフェース・データ構造と実装が一致している
3. **タスクの完了**: `tasks.md` の全タスクが実装済みかつテスト済み
4. **コード品質と堅牢性**: セキュリティ・エラーハンドリング・堅牢性に問題がない
5. **ステアリングとの整合**: `development-rules.md` の制約を違反していない

Claude Code が担うのはステップ3（spec の記述品質）とステップ6（コード品質・spec 整合性）。
Kiro はステップ7で spec 完了処理を行う。詳細は `docs/CONTRIBUTING.md` を参照。
