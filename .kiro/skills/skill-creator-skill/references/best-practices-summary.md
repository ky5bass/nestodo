# Agent Skills ベストプラクティス要約

本ドキュメントは `agent-skills-best-practices.md` の要点を、Skill生成時のクイックリファレンスとしてまとめたものである。各項目の詳細は元ドキュメントを参照すること。

---

## 1. ディレクトリ構成

| 構成 | 内容 |
|---|---|
| 最小構成 | `SKILL.md` のみ（必須） |
| 拡張構成 | `SKILL.md` + `scripts/` + `references/` + `assets/`（すべて任意） |

### 各ディレクトリの役割

| ディレクトリ | 役割 | ポイント |
|---|---|---|
| `scripts/` | 実行可能コード | 自己完結型、有用なエラーメッセージ、エッジケース処理 |
| `references/` | 追加ドキュメント | 焦点を絞った個別ファイル、オンデマンド読み込み |
| `assets/` | 静的リソース | テンプレート、画像、データファイル等 |

---

## 2. SKILL.md フォーマット

### frontmatter フィールド一覧

| フィールド | 必須 | 制約 |
|---|---|---|
| `name` | はい | kebab-case、最大64文字、親ディレクトリ名と一致 |
| `description` | はい | 最大1024文字、機能+使用タイミング+トリガーキーワード |
| `license` | いいえ | ライセンス名またはファイル参照 |
| `compatibility` | いいえ | 最大500文字、環境要件 |
| `metadata` | いいえ | 任意のキー・バリュー（author, version等） |
| `allowed-tools` | いいえ | スペース区切りの事前承認ツールリスト（実験的） |

### kebab-case ルール チェックリスト

- [ ] 小文字英数字（`a-z`, `0-9`）とハイフン（`-`）のみ
- [ ] ハイフンで開始しない
- [ ] ハイフンで終了しない
- [ ] 連続ハイフンを含まない
- [ ] 64文字以内
- [ ] 親ディレクトリ名と一致
- [ ] 正規表現: `^[a-z0-9]+(-[a-z0-9]+)*$`

### 推奨セクション構成

| セクション | 必須度 | 内容 |
|---|---|---|
| 概要 | 必須 | 目的を簡潔に |
| ワークフロー | 必須 | ステップバイステップの手順 |
| ルール | 必須 | 遵守すべきルール一覧 |
| 出力フォーマット | 推奨 | 期待する出力テンプレート |
| 例 | 推奨 | 入力例と出力例 |
| ガイドライン | 推奨 | 追加のガイドライン |

---

## 3. Progressive Disclosure（段階的開示）

| レベル | タイミング | 内容 | トークンコスト |
|---|---|---|---|
| メタデータ | 起動時 | name + description のみ | 約100トークン |
| 指示 | Skill発動時 | SKILL.md本文全体 | 5,000トークン未満推奨 |
| リソース | 必要時のみ | scripts/, references/, assets/ | 必要に応じて |

### 重要な制約

- SKILL.md本文は **500行以下** に抑える
- 詳細な参照資料は別ファイルに分離する
- 相互排他的なコンテキストは別ファイルに分離する

---

## 4. description フィールドの書き方

### 必須3要素

1. **何ができるか**: 機能を具体的に記述
2. **いつ使うべきか**: 使用タイミングを明示
3. **トリガーキーワード**: アクティベーション判定用キーワード

### 良い例・悪い例

| 種別 | 例 |
|---|---|
| ❌ 悪い例 | `Helps with PDFs.` |
| ✅ 良い例 | `Extracts text and tables from PDF files, fills PDF forms, and merges multiple PDFs. Use when working with PDF documents or when the user mentions PDFs, forms, or document extraction.` |

---

## 5. ファイル参照ルール

- SKILL.mdからの **相対パス** で参照する
- 参照は **1階層の深さ** まで（例: `references/guide.md` は可、`references/sub/guide.md` は不可）

---

## 6. 開発・評価のベストプラクティス

| 原則 | 内容 |
|---|---|
| 評価から始める | 代表的なタスクで実行し、不足箇所を特定してからSkillを構築 |
| スケールを意識 | SKILL.mdが大きくなったら別ファイルに分割 |
| エージェントの視点 | 実際のシナリオでの使われ方を監視し反復改善 |
| エージェントと反復 | 成功アプローチや間違いをSkillとしてキャプチャ |

---

## 7. セキュリティ注意事項

- [ ] 信頼できるソースからのみSkillをインストール
- [ ] 信頼度の低いソースは使用前に徹底的に監査
- [ ] バンドルされたファイル（特にスクリプト）の内容を確認
- [ ] 外部ネットワーク接続を含むコードや指示に注意

---

## 8. 配置場所

| スコープ | パス |
|---|---|
| グローバル | `~/.kiro/skills/<skill-name>/SKILL.md` |
| ワークスペース | `.kiro/skills/<skill-name>/SKILL.md` |

---

## 9. バリデーション

```bash
skills-ref validate ./my-skill
```

SKILL.mdのfrontmatterが有効で、すべての命名規則に従っているかをチェックする。
