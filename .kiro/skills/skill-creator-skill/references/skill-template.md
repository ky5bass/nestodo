# SKILL.mdテンプレート

> 本テンプレートは、Skill Creator Skillが生成するSKILL.mdの完全なテンプレートである。
> コピー＆ペーストで使用し、`{変数名}` のプレースホルダーを実際の値に置換して使用する。
> `<!-- ... -->` のコメントは記述ガイドであり、生成時に削除する。

---

## テンプレート本文

```markdown
---
name: "{skill-name}"
# 記述ガイド: kebab-case形式（小文字英数字+ハイフン）、最大64文字、配置先ディレクトリ名と一致させる
# 例: my-awesome-skill, code-review-helper

description: >
  {Skillの機能の具体的な説明}。
  使用タイミング: {いつこのSkillを使うべきかの説明}。
  トリガーキーワード（日本語）: {日本語キーワードをカンマ区切りで列挙}。
  トリガーキーワード（英語）: {英語キーワードをカンマ区切りで列挙}。
# 記述ガイド: 最大1024文字。以下の3要素を必ず含める。
#   1. 機能: Skillが何をするかの具体的な説明
#   2. 使用タイミング: いつこのSkillを使うべきか
#   3. トリガーキーワード: アクティベーション判定に使用するキーワード（日本語・英語）

metadata:
  version: "{version}"
  # 記述ガイド: セマンティックバージョニング推奨（例: "1.0", "1.1"）
  author: "{author}"
  # 記述ガイド: 作成者名またはチーム名
---

# {Skill名}
<!-- 記述ガイド: Skillの表示名。frontmatterのnameとは異なり、自然な名称を使用可能 -->

## 概要
<!-- 記述ガイド: Skillの目的と機能を2〜3文で簡潔に記述する -->
<!-- このセクションは必須 -->

{Skillの目的と主な機能を簡潔に記述する。何ができるか、どのような場面で役立つかを明確にする。}

## ワークフロー
<!-- 記述ガイド: Skillの実行手順をステップバイステップで記述する -->
<!-- このセクションは必須 -->
<!-- 複雑なワークフローはサブステップに分割する -->

### ステップ1: {ステップ名}

{手順の詳細を記述する。}

### ステップ2: {ステップ名}

{手順の詳細を記述する。}

### ステップ3: {ステップ名}

{手順の詳細を記述する。必要に応じてステップを追加・削減する。}

## ルール
<!-- 記述ガイド: Skillが遵守すべきルールを番号付きリストで記述する -->
<!-- このセクションは必須 -->
<!-- 各ルールは具体的かつ検証可能な形で記述する -->

1. {ルール1: 具体的な制約や条件を記述}
2. {ルール2: 具体的な制約や条件を記述}
3. {ルール3: 必要に応じてルールを追加する}

## 出力フォーマット
<!-- 記述ガイド: Skillが生成する成果物のテンプレートや形式を記述する -->
<!-- このセクションは推奨（省略可能） -->
<!-- コードブロックやテーブルを活用して具体的な形式を示す -->

{このSkillが生成する成果物のテンプレートや期待する出力形式を記述する。}

## 例
<!-- 記述ガイド: 入力例と出力例を記述する。エッジケースも含める -->
<!-- このセクションは推奨（省略可能） -->
<!-- 典型的なケースとエッジケースの両方を含めると効果的 -->

### 入力例

{ユーザーの入力例を記述する。}

### 出力例

{期待される出力例を記述する。}

### エッジケース

{特殊なケースや境界条件での動作例を記述する。}

## ガイドライン
<!-- 記述ガイド: 追加のガイドラインや注意事項を記述する -->
<!-- このセクションは推奨（省略可能） -->
<!-- ベストプラクティスやよくある間違いへの対処法を含める -->

- {ガイドライン1: 推奨事項や注意点を記述}
- {ガイドライン2: 推奨事項や注意点を記述}
- {ガイドライン3: 必要に応じてガイドラインを追加する}
```

---

## プレースホルダー一覧

| プレースホルダー | 説明 | 制約 |
|---|---|---|
| `{skill-name}` | Skill名（kebab-case） | 小文字英数字+ハイフン、最大64文字 |
| `{Skill名}` | Skillの表示名 | 自然な名称を使用可能 |
| `{Skillの機能の具体的な説明}` | Skillが何をするかの説明 | description内、具体的に記述 |
| `{いつこのSkillを使うべきかの説明}` | 使用タイミング | description内 |
| `{日本語キーワードをカンマ区切りで列挙}` | 日本語トリガーキーワード | description内 |
| `{英語キーワードをカンマ区切りで列挙}` | 英語トリガーキーワード | description内 |
| `{version}` | バージョン番号 | セマンティックバージョニング推奨 |
| `{author}` | 作成者名 | 任意の文字列 |
| `{ステップ名}` | ワークフローのステップ名 | 簡潔な名称 |
| `{ルールN}` | 遵守すべきルール | 具体的かつ検証可能 |
| `{ガイドラインN}` | 追加のガイドライン | 推奨事項や注意点 |

## セクション構成ガイド

| セクション | 必須 | 目的 |
|---|---|---|
| 概要 | はい | Skillの目的と機能の簡潔な説明 |
| ワークフロー | はい | ステップバイステップの実行手順 |
| ルール | はい | 遵守すべきルール一覧 |
| 出力フォーマット | 推奨 | 期待する出力のテンプレート |
| 例 | 推奨 | 入力例と出力例 |
| ガイドライン | 推奨 | 追加のガイドラインや注意事項 |

## 注意事項

- SKILL.md本文は500行以下に収める。超過する場合は `references/` 配下に詳細を分離する
- すべてのファイル参照はSKILL.mdからの相対パスで記述する（例: `references/guide.md`）
- ファイル参照の深さは1階層まで（例: `references/sub/guide.md` は不可）
- descriptionは1024文字以内とし、機能・使用タイミング・トリガーキーワードの3要素を必ず含める
- frontmatterのnameは配置先ディレクトリ名と一致させる
