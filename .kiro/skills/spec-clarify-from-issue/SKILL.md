# Issue を具体化して承認内容を記録する手順

この steering は、GitHub Issue の内容を spec に反映できる粒度までユーザーとの対話で具体化し、承認された内容を Issue に記録するときに使用する。このセッションでは spec の変更、ブランチ作成、PR 作成を行わない。

## 依頼形式

ユーザーは次のように依頼する:

```text
#spec-clarify-from-issue Issue #XX を具体化してください。
```

## 手順

1. `#github-operations` steering を参照する
2. Issue 本文とコメントを確認する
3. `docs/spec-index.md` と `.kiro/specs/` 配下を確認し、対象 spec と既存仕様を把握する。この時点では spec を編集しない
4. spec へ反映するために不足している判断、曖昧な点、内容の矛盾を整理する
5. 未確定事項がある場合は「対話による具体化ルール」に従ってユーザーへ質問し、回答に応じて次の質問を調整する
6. 未確定事項が解消したら「承認用 spec 反映案のルール」に従って対話内容を統合し、ユーザーへ承認を求める
7. 明示的な承認を得たら、「Issue への承認内容記録ルール」に従って Issue にコメントする
8. Issue 本文とコメントを再取得し、承認済みコメントが投稿されたことを確認する
9. spec を変更せずにセッションを終了し、新しいセッションで次を実行するようユーザーへ案内する。ユーザーから同セッション内で spec 反映を求められても応じず、セッション分離の理由（過去の会話ではなく Issue の承認済みコメントを唯一の正とするため）を伝えて断る

```text
#spec-update-from-approved-issue Issue #XX を spec に反映してください。
```

## 対話による具体化ルール

- 原則として一度に 1 つの論点を質問する。互いに強く関連し、簡単に回答できる論点は 2〜3 件までまとめてよい
- 白紙から考えさせる質問を避け、推奨案とその簡潔な理由を示す。選択肢が有効な場合は、推奨案を含む少数の選択肢を示す
- Issue 本文、Issue コメント、既存 spec から判断できる事項を重ねて質問しない
- ユーザーが判断する必要のない内部実装の詳細は質問せず、既存設計とプロジェクトルールから合理的に決める
- 最初に主な未確定論点を整理する。対話が長くなる場合は「残り 2 点」のように進行状況を伝える
- 回答によって不要になった質問は省略し、新たな論点が生じた場合はその理由を伝える
- 回答間に矛盾がある場合は、矛盾する箇所だけを示して確認する
- 未確定事項が残っている間は、全体の承認を求めない

## 承認用 spec 反映案のルール

- 質問と回答を時系列で転載せず、最終的に合意する内容として再構成する
- 過去の会話を参照しなくても理解できる、自己完結した内容にする
- 変更内容を正確に判断できるだけの情報を含め、機械的な文字数・行数制限は設けない
- 読む負荷を抑えるため、今回必要な項目だけを使って構造化する。項目の例は、目的、主な振る舞い、例外・エラー時の振る舞い、設計上の判断、対象外、対象 spec とする
- 既存 spec から変更しない事項や、ユーザー判断が不要な細かな内部実装は列挙しない
- 対象 spec は、既存更新・新規作成・アーカイブの種別を付けてリストで示す
- 「承認」「この内容で進めてください」など、反映案全体への明示的な同意だけを承認とみなす。修正指示、条件付き回答、質問、部分的な同意は承認とみなさない

## Issue への承認内容記録ルール

承認された spec 反映案を、`#github-operations` steering に従って次の形式で Issue にコメントする。質問と回答の全履歴は含めない。

```markdown
## 承認済み spec 反映内容

<!-- spec-update-approved:v1 -->

ユーザーの承認を受け、以下の内容を spec に反映します。

<!-- 承認された自己完結した spec 反映案 -->

<sub>Generated by Kiro</sub>
<!-- ai-agent:kiro -->
```

- 同じ Issue に承認済みコメントが複数ある場合は、最新のコメントを有効とする
- 承認後に内容を変更する場合は、変更点を含む反映案全体について再承認を得て、新しい承認済みコメントを投稿する
- 承認済みコメントの投稿後は、このセッションで spec 更新へ進まない
